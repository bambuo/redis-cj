# Redis 序列化协议规范 (RESP)

Redis 客户端使用一种称为 Redis 序列化协议（RESP）的协议与 Redis 服务器通信。
虽然该协议是为 Redis 专门设计的，但您也可以将其用于其他客户端-服务器软件项目。

RESP 在以下考虑因素之间取得了平衡：

* **易于实现** — 简单直观，降低开发门槛
* **快速解析** — 性能可媲美二进制协议
* **人类可读** — 便于调试和观察

RESP 可以序列化不同类型的数据，包括整数、字符串和数组，还包含专门的错误类型。
客户端将请求作为字符串数组发送给 Redis 服务器，数组内容为命令及其参数，服务器的回复类型因命令而异。

RESP 是**二进制安全**的，使用**前缀长度**来传输批量数据，因此无需对传输的批量数据进行额外处理。

> **注意**：此处概述的协议仅用于客户端-服务器通信。Redis 集群使用不同的二进制协议在节点之间交换消息。

---

## RESP 版本

RESP 协议的第一个版本在 Redis 1.2 中引入（可选使用）。
在 Redis 2.0 中，协议的下一代版本（即 RESP2）成为客户端与 Redis 服务器通信的标准方式。

[RESP3](https://github.com/redis/redis-specifications/blob/master/protocol/RESP3.md) 主要是 RESP2 的超集，旨在让客户端作者的工作更轻松。
Redis 6.0 引入了 RESP3 功能的实验性可选支持（不包括流式字符串和流式聚合类型）。
[`HELLO`](https://redis.io/commands/hello) 命令的引入允许客户端握手并升级连接的协议版本。

从 Redis 7.0 开始，RESP2 和 RESP3 客户端都可以调用所有核心命令，但命令可能会为不同的协议版本返回不同类型的回复。

---

## 网络层

客户端通过创建到 Redis 端口的 TCP 连接来连接到 Redis 服务器（默认端口为 6379）。
虽然 RESP 在技术上不特定于 TCP，但在 Redis 上下文中，该协议专用于 TCP 连接（或等效的面向流连接，如 Unix 套接字）。

---

## 请求-响应模型

Redis 服务器接受由不同参数组成的命令，处理命令后将回复发送回客户端。

这是最简单的模型，但存在一些例外：

* **Pipeline（流水线）**：客户端可以一次性发送多个命令，稍后等待回复。
* **Pub/Sub（发布/订阅）**：RESP2 连接订阅频道后，协议变为*推送*协议，服务端自动发送新消息。
* **MONITOR 命令**：将连接切换到临时的推送模式。
* **保护模式**：从非回环地址连接到保护模式下的 Redis 时，连接被拒绝并终止。
* **RESP3 Push 类型**：允许服务器随时向连接推送带外数据。

除这些例外情况外，Redis 协议是一个简单的请求-响应协议。

## RESP 协议描述

RESP 本质上是一个支持多种数据类型的序列化协议。在 RESP 中，数据的第一个字节决定了其类型。

**客户端-服务器通信方式：**
* 客户端以**批量字符串数组**的形式向 Redis 服务器发送命令。数组中的第一个（有时也是第二个）批量字符串是命令名称，后续元素是命令参数。
* 服务器以 RESP 类型回复，回复类型由命令的实现决定，也可能取决于客户端的协议版本。

RESP 是一个使用标准 ASCII 编码控制序列的二进制协议。`\r\n` (CRLF) 是协议的**终止符**，**始终**用于分隔各部分。

RESP 数据类型分为三类：**简单类型**（类似标量值）、**批量类型**（二进制安全字符串）、**聚合类型**（包含子元素和嵌套层级）。

### RESP 数据类型汇总表

| RESP 数据类型 | 最低协议版本 | 分类 | 首字节 |
|-------------|-----------|------|-------|
| 简单字符串 (Simple strings) | RESP2 | 简单 | `+` |
| 简单错误 (Simple Errors) | RESP2 | 简单 | `-` |
| 整数 (Integers) | RESP2 | 简单 | `:` |
| 批量字符串 (Bulk strings) | RESP2 | 聚合 | `$` |
| 空批量字符串 (Null bulk strings) | RESP2 | 聚合 | `$-1\r\n` |
| 数组 (Arrays) | RESP2 | 聚合 | `*` |
| 空值 (Nulls) | RESP3 | 简单 | `_` |
| 布尔值 (Booleans) | RESP3 | 简单 | `#` |
| 双精度浮点数 (Doubles) | RESP3 | 简单 | `,` |
| 大数 (Big numbers) | RESP3 | 简单 | `(` |
| 批量错误 (Bulk errors) | RESP3 | 聚合 | `!` |
| 逐字字符串 (Verbatim strings) | RESP3 | 聚合 | `=` |
| 映射 (Maps) | RESP3 | 聚合 | `%` |
| 属性 (Attributes) | RESP3 | 聚合 | `\|` |
| 集合 (Sets) | RESP3 | 聚合 | `~` |
| 推送 (Pushes) | RESP3 | 聚合 | `>` |

---

### 简单字符串 (Simple Strings)

编码格式：`+` 字符后跟字符串，以 CRLF 终止。字符串不能包含 `\r` 或 `\n`。

示例：`+OK\r\n`（5 字节）

客户端应返回 `+` 之后到 CRLF 之前的字符串内容。如需传输二进制字符串，请使用批量字符串。

---

### 简单错误 (Simple Errors)

与简单字符串类似，但首字节为 `-`。客户端应将错误视为异常。

格式：`-错误消息\r\n`

示例：
```
-ERR unknown command 'asdf'
-WRONGTYPE Operation against a key holding the wrong kind of value
```

`-` 后的第一个大写单词（到第一个空格或换行为止）表示错误前缀（如 `ERR`、`WRONGTYPE`），客户端可用其区分错误类型。

---

### 整数 (Integers)

CRLF 终止的字符串，表示有符号的十进制 64 位整数。

格式：`:[<+|->]<值>\r\n`

示例：`:0\r\n`（0）, `:1000\r\n`（1000）

某些情况下，整数可表示布尔值（例如 `SISMEMBER` 返回 1 表示 true，0 表示 false）。

---

### 批量字符串 (Bulk Strings)

表示单个二进制字符串。默认最大 512 MB。

格式：`$<长度>\r\n<数据>\r\n`

示例：
```
$5\r\nhello\r\n     → "hello"
$0\r\n\r\n          → 空字符串
```

---

### 空批量字符串 (Null Bulk Strings)

RESP2 中表示不存在的值。编码为 `$-1\r\n`。

例如，当 `GET` 命令的键不存在时返回此类型。客户端应返回 `nil`/`NULL` 而非空字符串。

---

### 数组 (Arrays)

格式：`*<元素数量>\r\n<元素1>...<元素n>`

示例：
```
*0\r\n                              → 空数组
*2\r\n$5\r\nhello\r\n$5\r\nworld\r\n → ["hello", "world"]
*-1\r\n                             → 空数组（null）
```

数组支持嵌套和混合类型。聚合类型都支持嵌套。

---

### 空值 (Nulls) — RESP3

表示不存在的值。编码为 `_\r\n`。

RESP3 引入此类型来修复 RESP2 中 null 值双重表示的问题。

---

### 布尔值 (Booleans) — RESP3

格式：`#<t|f>\r\n`

`#t\r\n` = true, `#f\r\n` = false

---

### 双精度浮点数 (Doubles) — RESP3

格式：`,[<+|->]<整数>[.<小数>][<E|e>[符号]<指数>]\r\n`

示例：
```
,1.23\r\n
,inf\r\n      → 正无穷
,-inf\r\n     → 负无穷
,nan\r\n      → 非数值
```

---

### 大数 (Big numbers) — RESP3

格式：`([<+|->]<数字>\r\n`

示例：
```
(3492890328409238509324850943850943825024385\r\n
```

---

### 批量错误 (Bulk Errors) — RESP3

组合了简单错误的用途和批量字符串的表达能力。

格式：`!<长度>\r\n<错误>\r\n`

示例：
```
!21\r\n
SYNTAX invalid syntax\r\n
```

---

### 逐字字符串 (Verbatim Strings) — RESP3

类似于批量字符串，但额外提供了数据编码提示。

格式：`=<总长度>\r\n<3字节编码>:<数据>\r\n`

示例：
```
=15\r\n
txt:Some string\r\n
```

前 3 字节为编码类型（如 `txt` 纯文本、`mkd` Markdown），第 4 字节为 `:`。

---

### 映射 (Maps) — RESP3

编码键值对集合（字典/哈希表）。

格式：`%<条目数>\r\n<键1><值1>...<键n><值n>`

示例：
```
%2\r\n
+first\r\n
:1\r\n
+second\r\n
:2\r\n
```

> **RESP2 兼容**：RESP2 没有映射类型，使用扁平数组 `[k1, v1, k2, v2, ...]` 表示。

---

### 属性 (Attributes) — RESP3

与映射完全相同，但首字节为 `|`。属性不是回复的一部分，而是辅助数据。

客户端遇到属性时应读取属性并保存，然后继续读取真正的回复。

---

### 集合 (Sets) — RESP3

类似于数组，但无序且应只包含唯一元素。

格式：`~<元素数>\r\n<元素1>...<元素n>`

---

### 推送 (Pushes) — RESP3

包含带外数据。不同于请求-响应模型，提供通用的推送模式。

格式：`><元素数>\r\n<元素1>...<元素n>`

推送数据可出现在任何 RESP 数据之前或之后（但不会出现在其内部）。客户端应通过回调处理推送数据。

---

## 客户端握手 (Client Handshake)

新的 RESP 连接应首先调用 `HELLO` 命令：
* 允许服务器向后兼容 RESP2 版本
* 返回服务器和协议信息供客户端使用

语法：`HELLO <协议版本> [AUTH <用户名> <密码>]`

默认连接从 RESP2 模式开始。如果协议版本不受支持，服务器返回 `-NOPROTO` 错误。

成功回复为一个 Map，包含以下字段（部分为 Redis 特有）：
* **server**: "redis"（或其他软件名称）
* **version**: 服务器版本
* **proto**: 支持的最高 RESP 协议版本
* **id**: 连接标识符
* **mode**: "standalone"/"sentinel"/"cluster"
* **role**: "master"/"replica"
* **modules**: 已加载模块列表

## 发送命令到 Redis 服务器

客户端发送由**仅包含批量字符串的数组**组成的命令，服务器回复任何有效的 RESP 数据类型。

示例 — `LLEN mylist`：
```
C: *2\r\n
C: $4\r\n
C: LLEN\r\n
C: $6\r\n
C: mylist\r\n
S: :48293\r\n
```

---

## 多命令与流水线 (Pipelining)

客户端可以使用同一连接发出多个命令。支持流水线，多个命令可通过一次写操作发送，所有回复可在最后一次性读取。

---

## 内联命令 (Inline Commands)

当只有 telnet 可用时，Redis 也接受内联命令格式。参数以空格分隔，无需以 `*` 开头。

示例：
```
C: PING
S: +PONG

C: EXISTS somekey
S: :0
```

---

## 高性能 RESP 解析器

RESP 使用前缀长度传输批量数据，无需扫描特殊字符（如 JSON），也无需引号和转义。

示例（C 语言）：
```c
unsigned char *p = "$123\r\n";
int len = 0;
p++;
while(*p != '\r') {
    len = (len*10)+(*p - '0');
    p++;
}
// len = 123
```

在大多数高级语言中，实现 RESP 协议比实现等效的二进制协议更直接，减少了客户端软件中的 Bug。
